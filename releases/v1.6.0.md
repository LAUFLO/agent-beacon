# Agent Beacon v1.6.0

## 更新内容

- 新增多任务状态模型。同一 Agent 的等待任务不会再被较新的运行任务覆盖，桌面灯按“等待处理 > 正在运行 > 已结束”汇总。
- 灯杆在原有高度内新增独立状态中心按钮，同时保留设置和关闭按钮；三个按钮缩小并等距排列，状态中心按钮直接打开新总览，不增加常驻计数牌。
- 当前任务支持按项目目录、会话短 ID 和窗口标题进行优先定位；无法精确匹配时仍安全回退到对应 Agent 窗口。
- 新增未解决黄灯恢复。Agent Beacon 重启或自动更新后仍未解决的等待任务会恢复，并只提醒一次。
- 恢复快照只保存状态所需的匿名任务标识与时间，不保存任务标题、聊天正文或项目路径。
- 新增状态源可信度，区分正常、等待事件、未配置、已失联和数据陈旧；不可信状态会在灯头显示灰色标记，并在状态中心说明原因。
- 新增可信阶段与进度显示。只展示 Agent 实际上报的阶段和百分比，不推测进度。
- 运行任务超过 10 分钟没有进展事件时标记为“可能卡住”，保持运行状态，不错误显示为完成。
- Claude Code Hooks 和 OpenCode 插件上报事件类型、阶段、项目目录与最后活动时间；TRAE MCP 保持原有精简工具参数和 `ok` 返回值，不增加固定 Token 开销。
- 状态历史、通知去重和今日统计改为按任务 ID 处理，支持同一 Agent 多个并行任务。
- 当前任务浮窗支持拖动；设置、状态中心、通知策略、更新进度、安装和应用内确认窗口均可自由移动。
- 设置页高度由 500 像素压缩到 438 像素，今日统计改为单行紧凑区域，去除底部重复说明和大块留白。
- 状态中心改为四 Agent 概览卡、待处理列表和运行列表组成的总览首页；新增“更多 / 返回”双向切换，原状态源诊断、历史、复制和清空功能完整保留在更多页。
- 修复 Codex 界面审批兜底每次扫描生成新任务 ID 的问题；临时审批不再持久化，同一会话的新运行事件会覆盖旧确认状态。
- 修复 Codex 多会话确认识别只检查最后更新会话的问题；一个会话等待确认、另一个继续运行时，状态中心保留两项任务并按黄灯汇总。
- 修复 Codex 完成后仍显示绿色的问题：同一会话先按最新事件折叠，新的完成态覆盖旧运行态；完成后的 Token 统计与响应尾部事件不再生成幽灵运行任务。
- Claude Code 多会话时停用无法区分会话的全局工具进程覆盖，避免其他会话运行工具时把等待确认任务误改为绿色；TRAE 与 OpenCode 继续按独立会话事件聚合。
- 状态中心的待处理任务新增“清除”操作，可忽略 Agent 未补报结束状态而遗留的黄灯；清理结果会持久化，同一任务出现更新事件时自动解除忽略并继续监控。
- 状态中心首页最多直接展示 3 条等待任务，修复等待总数与可见行数不一致；新增“清除全部”，可一次清理 TRAE 规则未补报收尾而残留的多条黄灯。
- 修复“3 条等待、1 条运行”等任务组合打开状态中心时可能发生的列表索引越界；状态扫描继续实时运行，页面改为内容变化时一次性重绘，消除定时刷新闪烁。
- 同一项目有更新的 Codex 会话接替运行时，自动移除超过 10 分钟无进展的旧会话，避免一个真实任务被重复计数；仍持续更新的并行任务不会被合并。
- 当前任务浮层改为内容签名增量刷新，内容不变时不重建控件，消除每次扫描造成的列表闪烁。
- 状态中心压缩为与设置页一致的 `760×438`，内容页四周保留 6 像素边距，恢复完整粗像素外框；其余应用弹窗均不超过该尺寸。
