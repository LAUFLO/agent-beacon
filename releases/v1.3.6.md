# Agent Beacon v1.3.6

## 更新内容

- 收紧 TRAE MCP 的显示时序：准备确认消息、执行延时或其他前置工作时继续保持 `running`，只有下一步立即显示确认时才上报 `waiting`。
- 只有全部工作结束且最终答复已经完整准备后，才把 `completed` 作为最后一次工具调用并立即输出答复。
- `waiting` 和 `completed` 的返回值分别附带极短的“立即显示确认”和“立即输出答复”提醒，使黄灯和红灯更贴近真实界面时刻。
- 保持 TRAE 纯 MCP 模式，不恢复日志、Work 会话、UIAutomation 或截图兜底。

## 升级说明

升级后请在 Agent Beacon 设置中重新复制 TRAE 状态规则，并覆盖 TRAE 中原有的全局规则。MCP 命令路径保持不变。

如果 Helper 正被 TRAE 占用，请完全退出 TRAE，启动新版 Agent Beacon 完成 Helper 替换，然后重新打开 TRAE。
