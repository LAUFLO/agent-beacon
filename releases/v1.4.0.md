# Agent Beacon v1.4.0

## 更新内容

- TRAE MCP Helper 新增握手、活动与断开记录；设置页显示最近通信时间，10 分钟没有有效通信后显示“已失联”。TRAE 继续保持仅 MCP 模式。
- 新增 GitHub Release 自动更新：后台检查版本，下载后校验 SHA-256 和 EXE 内部版本，退出旧进程后原子替换并重新启动。
- 新增黄灯和红灯 Windows 通知，可分别开关、按 TRAE/Codex/Claude Code/OpenCode 单独启用，并支持 22:00–08:00 免打扰。
- 新增点击状态灯定位 Agent：单击桌面灯头或双击任务栏状态灯即可切换窗口。
- 新增本机轻量状态历史，只记录时间、Agent、颜色、说明与判定来源，不保存任务标题、会话 ID 或聊天正文，并自动限制文件大小。
- 设置页、状态中心、右键菜单和应用内提示弹窗统一改为白底重像素风格；全部应用内文字使用统一中文像素字体，适合居中的标题、短提示和状态文字均已居中。
- 设置、状态中心、更新提示、更新进度和主窗口统一使用 EXE 内嵌的 Agent Beacon 图标，避免 Dock 或任务栏显示系统默认程序图标。
- Agent Beacon Logo 重绘为纯色硬边的重像素风格，统一用于 README、EXE 文件、Dock/任务栏窗口及全部应用窗口；ICO 内置 16–256 像素的九档清晰尺寸。
- 自动更新新增分段像素进度条，显示下载、SHA-256 校验、版本校验和准备安装阶段；状态中心固定尺寸，将实时诊断与状态历史合并，并使用支持滚轮、翻页和拖动的自绘像素滚动条。
- 将版本信息、状态历史、桌面功能、界面、Agent 集成和更新服务拆分为独立源码模块，降低后续功能之间的耦合。
- 新增统一构建脚本和 GitHub Actions：自动执行状态回放、MCP 协议、集成事件和发布流程测试，生成 EXE 与 SHA-256，并在推送 `v*` 标签时创建 GitHub Release。

## 升级说明

TRAE MCP 命令路径保持不变。升级并启动 v1.4.0 后，Agent Beacon 会按内容哈希自动更新 Helper；如果 Helper 正被 TRAE 占用，请完全退出 TRAE，再重新启动 Agent Beacon 完成替换。

自动更新通过 GitHub Release 元数据、SHA-256 和内部版本号进行校验。
